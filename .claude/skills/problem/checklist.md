# Problem Capture Checklist

Before saving, verify every item is filled in the output file.

## Required

- [ ] **Title** — short, specific, searchable (not "bug" or "issue")
- [ ] **Problem statement** — one clear paragraph; what is wrong or blocked
- [ ] **Error or unexpected behavior** — exact error message, output, or description of what went wrong
- [ ] **What was tried** — at least one attempted approach and why it failed (write "nothing yet" if nothing was tried)
- [ ] **Status** — one of: `open`, `blocked`, `needs-research`

## Optional but Useful

- [ ] **Relevant code snippets** — paste only the minimal relevant section
- [ ] **Notes for next session** — any hunches, links, or hints for whoever picks this up
- [ ] **Tags** — e.g., `azure`, `auth`, `csharp`, `openai`
